# Система автоматического отслеживания новых версий программного обеспечения

## Описание проекта

Система предназначена для мониторинга и отслеживания новых версий программного обеспечения в режиме реального времени. Приложение позволяет:

- **Добавлять проекты** для отслеживания версий
- **Мониторить обновления** с использованием GitHub API и других источников
- **Получать уведомления** о выходе новых версий
- **Просматривать историю** обновлений в удобном веб-интерфейсе
- **Анализировать тренды** развития проектов

## Анализ предметной области

### Актуальность
В современной разработке ПО критически важно отслеживать обновления зависимостей и утилит. Автоматизация этого процесса позволяет:
1. Своевременно получать информацию об обновлениях безопасности
2. Планировать обновления проектов
3. Анализировать статус разработки проектов-зависимостей
4. Выявлять уязвимости через обновления

### Целевая аудитория
- Разработчики и команды разработки
- DevOps инженеры
- Менеджеры проектов
- Системные администраторы

## Выбор технологий

### Backend: Python + Flask
**Причины выбора:**
- Простота и быстрота разработки
- Обширная экосистема библиотек (requests, pyquery, sqlalchemy)
- Отличная поддержка REST API
- Хорошая документация и сообщество
- Лицензия: BSD (OSS совместима)

### Frontend: HTML5 + CSS3 + Vanilla JavaScript
**Причины выбора:**
- Не требуется npm и сложная сборка
- Быстрая загрузка
- Простоя интеграция с backend
- Bootstrap для стилизации (MIT лицензия)

### База данных: SQLite (разработка) / PostgreSQL (production)
**Причины выбора:**
- SQLite: простая настройка для разработки
- PostgreSQL: масштабируемость, надёжность (лицензия PostgreSQL - permissive OSS)

### Интеграции
- **GitHub API v3**: Для отслеживания версий репозиториев
- **PyPI API**: Для мониторинга Python-пакетов

## Установка и запуск

### Требования
- Python 3.9 или выше
- pip (Python Package Manager)

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: `http://localhost:5000`

### Переменные окружения

Создайте файл `.env`:

```
GITHUB_TOKEN=your_github_token_here
DEBUG=True
DATABASE_URL=sqlite:///version_tracker.db
```

## API Endpoints

### Projects (Проекты)
- `GET /api/projects` - Список всех проектов
- `POST /api/projects` - Создать новый проект
- `GET /api/projects/<id>` - Получить информацию о проекте
- `PUT /api/projects/<id>` - Обновить проект
- `DELETE /api/projects/<id>` - Удалить проект

### Versions (Версии)
- `GET /api/projects/<id>/versions` - Получить версии проекта
- `GET /api/projects/<id>/latest-version` - Получить последнюю версию

### Updates (Обновления)
- `POST /api/projects/<id>/check-update` - Принудительно проверить обновление
- `GET /api/updates/history` - История обновлений
